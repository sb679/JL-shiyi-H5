


### 一、项目概述

| 项目名称 | JL拾遗（校园二手书交易平台） |
|---------|---------------------|
| 版本 | v1.0 MVP（最简可行产品） |
| 目标用户 | 本校全体学生（预计不超过10万人） |
| 核心价值 | 让二手书交易从"群聊刷屏、信息混乱"变为"结构化展示、按需检索、状态清晰" |
| 技术约束 | 微信小程序 + 云开发，**不支持在线支付**，交易通过线下完成，小程序仅作为信息撮合平台 |
| 非目标 | v1 不做即时通讯（IM）、不做在线支付、不做物流跟踪 |

### 关键风险（已识别并给出缓解方案）

| 风险 | 影响 | 缓解方案 |
|------|------|---------|
| 恶意用户批量点击"决定购买"导致正常书籍下架 | 破坏信任、交易无法完成 | 改为"卖家确认售出"机制（见4.3.3） |
| 联系方式明文公示 | 用户隐私泄露、骚扰风险 | v1 默认隐藏，仅"决定购买"后交换（见4.3.3） |
| 云开发数据库不支持全文检索，模糊搜索性能差 | 搜索体验差、超时 | v1 使用 `db.RegExp` + 前端防抖 + 限制搜索次数；后续考虑 Elasticsearch 或外部搜索服务 |
| ISBN 第三方 API 不稳定 | 自动补全失败、用户体验差 | 提供手动录入兜底，API 调用超时 3s，失败后静默降级 |
| 图片上传无限制导致云存储费用暴增 | 运营成本高 | 前端压缩（最大 2MB/张）+ 云函数端二次校验 |

### 二、用户痛点（已整理为验收标准）

| 编号 | 痛点描述 | 对应的解决方案 |
|-----|---------|--------------|
| P1 | 群聊里卖书只拍书脊侧面，不输书名，找书困难 | 强制结构化录入：书名/ISBN 为必填项 |
| P2 | 照片拍不到 ISBN，版本对不上，买错书 | 接入公开书籍 API（如 ISBN 查询），自动补全书籍信息 |
| P3 | 买卖信息在群里被刷掉，只有最新消息才有曝光 | 书籍永久上架，按发布时间/价格/热度排序，支持搜索 |
| P4 | 书籍卖掉了，原照片无法撤回，信息不同步 | 支持卖家确认"已售出"，书籍状态实时更新，已售书籍自动进入"售罄"区域，不再显示在售列表中 |

### 三、用户角色与权限

| 角色 | 权限描述 |
|-----|---------|
| **普通用户（学生）** | 浏览书城、搜索书籍、查看详情、发布书籍、编辑/下架自己发布的书籍、对感兴趣的书籍表达"想买"意向、作为卖家确认售出并标记售罄、对已完成交易的书籍进行双向评价 |
| **管理员（后续版本）** | 处理违规书籍、封禁用户（v1 暂不实现，可留手动后台接口） |

> v1 不区分"买家/卖家"身份，任何用户均可发布和购买。
>
> **交易流程**：买家表达"想买"意向 → 双方线下联系/交易 → 卖家在应用中确认"已售出" → 书籍状态变为"售罄"。
> 同一本书**允许多人表达"想买"意向**，但只有卖家可以最终确认售出。这样避免了恶意用户直接标记售罄的滥用风险。

### 四、功能需求（详细）

#### 4.1 登录功能

- **实现方式**：微信小程序 `wx.login` + 云开发自动鉴权（无需用户手动输入任何信息）
- **首次登录**：自动在云数据库 `users` 集合中创建用户记录（字段：`_openid`, `nickname`（默认为"拾遗用户"）, `avatarUrl`（默认空）, `createTime`）
- **无需手机号绑定、无需学生认证**（v1 简化，后续视需要增加）
- **⚠ 当前限制**：微信已废弃 `wx.getUserInfo` 自动获取头像和昵称的能力（2021年4月起），因此所有用户默认使用"拾遗用户"和默认头像。v1.1 计划在"自己的书"页面添加用户自定义头像/昵称功能（`button open-type="chooseAvatar"` + `input type="nickname"`）

#### 4.2 底部导航栏（两个 Tab）

| Tab 名称 | 页面路径 | 功能描述 |
|---------|---------|---------|
| **书城** | `pages/bookmall/bookmall` | 展示所有在售书籍列表，支持搜索、筛选、排序 |
| **自己的书** | `pages/mybooks/mybooks` | 展示当前用户发布的书籍（分 Tab：在售中 / 已售罄），以及"发布新书"入口 |

#### 4.3 书城页面

**4.3.1 书籍列表展示**
- 每本书以**卡片**形式展示，包含：
  - 封面图（用户上传的第一张图，或从 API 获取的官方封面）
  - 书名
  - 作者（如有）
  - 价格（用户填写的售价）
  - 新旧程度标签
  - 分类标签（教材 / 小说 / 教辅 / 其他）
  - 发布者昵称（脱敏，如"张**"）
  - **状态标签**：仅"在售"状态显示绿色"在售"，"售罄"状态显示灰色"售罄"
- 列表默认按**发布时间倒序**（最新的在上）
- 支持上拉加载更多（分页，每次 20 条）
- 切换排序条件时，重置分页，从第 1 页重新加载

**4.3.2 搜索与筛选**
- **搜索框**：支持按书名、作者、ISBN 关键词模糊搜索（使用云数据库 `db.RegExp`，前端输入 300ms 防抖）
- **筛选栏**：
  - 按分类：教材 / 小说 / 教辅 / 其他
  - 按价格排序（从低到高 / 从高到低）
  - 按发布时间排序（最新 / 最旧）
- **性能注意**：搜索前必须选择分类作为前置过滤，避免全表正则扫描。切换筛选/排序时重置分页

**4.3.3 书籍详情页**
- 展示完整信息：
  - 封面图（多张，用户上传的可左右滑动）
  - 书名、作者、ISBN（如有）、分类
  - 价格、新旧程度（用户填写）
  - 发布者描述（自定义）
  - 发布者信息：昵称（脱敏，如"张**"）、发布时间
- **联系方式展示策略（隐私保护）**：
  - 默认不显示完整联系方式，仅显示提示"表达想买意向后可查看"
  - 用户点击"我想买"后，通过云函数返回发布者的联系方式（微信号/手机号），前端展示
  - 联系方式暂不脱敏存储，但不在前端直接明文暴露在列表中
- **"我想买"按钮**：
  - 仅在书籍状态为"在售"时显示
  - 点击后：
    - 在 `books` 集合的 `interestedUsers` 数组中记录当前用户 openid 和时间
    - 向发布者发送微信订阅消息通知（需用户授权订阅，模板另行申请）
    - 显示发布者联系方式
  - 允许多人表达"想买"意向，不改变书籍状态
- **"确认售出"按钮**（仅对发布者展示）：
  - 仅发布者可见（`_openid === 当前用户 openid`），且书籍状态为"在售"
  - 点击后弹出选择器：从所有"想买"用户中选择最终成交的买家
  - 确认后：
    - 书籍 `status = 'sold'`
    - 记录 `buyerOpenid` 和 `purchaseTime`
    - 书籍从在售列表移除，进入"已售罄"区域
- **评价入口**（仅对购买者和发布者展示，售罄后）：
  - 双方均可发起评价：
    - **购买者**：对书籍/卖家打分（1-5星）+ 文字评论（选填）+ 是否"交易顺利"
    - **发布者（卖家）**：对买家的信用反馈（"爽快交易"/"放鸽子"等标签）
  - 评价存储于 `evaluations` 独立集合（而非 books 的嵌套字段），支持多次交易记录
  - 评价后在书籍详情页可按时间线展示
  - **隐私设计**：评价列表仅展示评价者角色（"买家"/"卖家"），不展示 openid 或昵称

- **交易留言板**（v1 新增功能）：
  - **可见性**：仅卖家或已表达"想买"意向的买家可见
  - **功能**：
    - 文字留言（最多 500 字）
    - 气泡式消息展示（自己/对方左右分列）
    - 每笔交易最多 50 条留言
    - 发送留言后同步更新 `books.lastMessageTime`
  - **关闭条件**：书籍售罄/确认售出后留言板关闭
  - **提示**："自己的书"页面在售书籍若有留言，显示"· 有留言"提示

- **举报入口**（v1 新增功能）：
  - 详情页底部极隐式链接：灰色极小字"举报此书"
  - 弹窗选择举报原因：虚假信息 / 诈骗 / 违禁品 / 骚扰 / 其他
  - 可补充详情描述（最多 200 字）
  - 同一人对同一目标仅可举报一次
  - 举报写入 `reports` 集合（status='pending'），v1 不做自动处理，管理员手动查看

#### 4.4 "自己的书"页面

**4.4.1 顶部区域**
- 显示用户头像、昵称
- **"+ 发布新书"按钮**（固定位置）

**4.4.2 书籍列表（分两个 Tab）**
- **Tab 1：在售中** — 展示当前用户发布的、状态为 `status = 'available'` 的书籍
  - 每本书展示"想买人数"角标
  - 每本书支持：编辑（部分字段）、**下架**（软删除：`status = 'removed'`，需二次确认）
- **Tab 2：已售罄** — 展示当前用户发布的、状态为 `status = 'sold'` 的书籍
  - 仅展示，不可再编辑或删除（保留交易记录供查询）
  - 可看到购买者的昵称（脱敏）和购买时间
  - 如有对方评价，在此展示评价摘要

**4.4.3 发布新书流程**

**步骤 1：选择录入方式**
- **方式A：通过 ISBN 自动录入（推荐）**
  - 用户输入或扫码（调用 `wx.scanCode` 扫描条形码，仅支持 EAN_13 格式）录入 ISBN 号码
  - 调用公开 API 获取书籍信息（**推荐首选** `openlibrary.org` → 免费、无需 key、CORS 友好；备选 `api.jike.xyz` 需申请 key）
  - 请求超时 3s，失败后静默降级到手动录入
  - 自动填充：书名、作者、封面图 URL、出版社（需下载封面并上传至云存储）
  - 用户只需补充价格、新旧程度、联系方式等交易信息
- **方式B：手动录入（当 ISBN 查询不到或扫码失败时）**
  - 必填：书名、作者（或"未知"）、价格、分类、至少一张封面照片
  - 选填：ISBN、新旧程度、详细描述

**步骤 2：上传图片**
- 至少上传 1 张图片（建议封面或背面，**不建议**在图片中暴露个人联系方式）
- 最多上传 5 张
- 前端使用 `wx.compressImage` 压缩至最长边 ≤ 1920px、质量 80%，单张上限 2MB
- 上传至云存储路径 `books/{timestamp}_{random}.jpg`（避免暴露 openid）
- 返回 `fileID` 存入数据库

**步骤 3：填写交易信息**
- **价格**（必填，数字，单位元）
- **新旧程度**（必填，选项：全新 / 几乎全新 / 有轻微笔记 / 较旧）
- **联系方式**（必填，填写微信号或手机号，不会公开显示在列表中，仅"想买"用户可查看）
- **自定义描述**（选填，textarea）

**步骤 4：提交**
- 点击"发布"按钮，数据通过**云函数**写入云数据库 `books` 集合（避免前端直接写库绕过校验），字段包含：
  - `_openid`（发布者 openid，由云函数从 context 注入）
  - `title`, `author`, `isbn`, `category`, `coverImage`（第一张图作为封面）, `images`（数组，所有图片 fileID）
  - `price`, `condition`, `contact`, `description`
  - `status` = `'available'`
  - `createTime`（云函数服务器时间 `db.serverDate()`）
  - `interestedUsers` = `[]`（初始为空数组）
  - `soldInfo`（售罄后填充：`buyerOpenid`, `purchaseTime`）

### 七、非功能需求

- **并发量**：支持全校 10 万人同时访问，日常同时在线 1000 人以内（云开发免费套餐 + 按量付费足够）
- **响应时间**：列表页加载 ≤1.5 秒，详情页 ≤1 秒
- **数据安全**：仅发布者可编辑/下架自己的书籍；普通用户无法修改他人书籍；`contact` 字段不直接返回到前端列表，仅通过云函数鉴权后返回
- **图片审核**：v1 暂不接入内容安全 API，但发布时前端提醒"不得上传违规图片"；后续通过云函数调用微信 `security.imgSecCheck`
- **图片成本控制**：前端压缩（最长边 ≤1920px）+ 云函数端二次校验大小，单张上限 2MB
