# H5 产品需求文档

## 版本历史
| 版本 | 日期 | 修改人 | 修改内容 |
|------|------|--------|----------|
| v1.0 | 2026-06-08 | AI | 基于原微信小程序文档重写为 H5 网页版本 |

## 1. 项目概述
| 项目 | 内容 |
|------|------|
| 项目名称 | JL拾遗（校园二手书交易 H5 平台） |
| 版本 | v1.0 MVP |
| 目标用户 | 本校学生，移动端优先，兼容桌面端 |
| 核心价值 | 让二手书交易从群聊刷屏变为结构化发布、搜索、沟通、成交记录和评价 |
| 技术约束 | H5 网页 + 后端 API + 数据库 + 对象存储 |
| 交易边界 | 平台只做信息撮合，不做在线支付和担保交易 |

## 2. 用户痛点与解决方案
| 编号 | 痛点 | H5 解决方案 | 验收标准 |
|------|------|-------------|----------|
| P1 | 群聊信息混乱，书名不全 | 发布时强制结构化字段 | 没有书名、分类、价格、联系方式、图片时不能发布 |
| P2 | ISBN 和版本不清楚 | ISBN 查询自动补全，失败可手动录入 | 输入合法 ISBN 后 3 秒内返回结果或提示手动录入 |
| P3 | 信息很快被刷掉 | 书城持续展示、搜索、筛选、排序 | 默认展示在售书籍，支持分页和多条件筛选 |
| P4 | 卖掉后状态不同步 | 卖家确认售出，售罄书籍从在售列表移除 | 只有发布者能确认售出，确认后状态为 sold |
| P5 | 联系方式公开导致骚扰 | 默认隐藏联系方式，表达想买后鉴权返回 | 列表和详情初始接口均不返回完整联系方式 |
| P6 | 线下交易缺乏信任 | 售后双向评价、举报机制 | 已售书籍的买卖双方可各评价一次 |

## 3. 用户角色与权限
| 角色 | 权限 |
|------|------|
| 游客 | 浏览公开书城、搜索筛选、查看书籍基本详情；不能发布、想买、查看联系方式、留言、评价 |
| 登录用户 | 发布书籍、表达想买、留言、查看已授权联系方式、评价交易、举报 |
| 卖家 | 管理自己发布的书籍、下架在售书籍、选择最终买家并确认售出 |
| 买家 | 对感兴趣书籍表达想买、查看联系方式、留言、在成交后评价卖家 |
| 管理员 | 后续版本处理举报、下架违规书籍、封禁账号、查看审计日志 |

## 4. 账号与登录需求
### 4.1 登录方式
MVP 推荐采用验证码登录，支持以下任一方式落地：
- 手机号 + 短信验证码：适合移动端，但有短信成本。
- 邮箱 + 验证码：成本低，后续可限制学校邮箱后缀。

若暂时无法接入短信或邮件服务，可先在开发环境使用演示验证码，但生产环境不得使用固定万能验证码。

### 4.2 会话策略
- 登录成功后服务端创建会话并返回 HttpOnly Secure Cookie，或返回短期 access token + refresh token。
- 推荐 Cookie Session，减少前端暴露 token 的风险。
- 会话有效期：7 天；敏感操作可要求最近登录时间不超过 30 天。
- 退出登录后服务端作废 refresh token 或 session。

### 4.3 用户资料
字段包括：昵称、头像、校区、学院、专业、联系方式偏好、创建时间、最近活跃时间。

验收标准：
- 新用户首次登录后自动创建用户记录。
- 昵称默认值为“拾遗用户”加短随机后缀，避免全站同名。
- 用户可在个人页修改昵称、头像、校区、学院、专业。
- 头像上传必须限制格式和大小。

## 5. 功能需求
### 5.1 书城首页
#### 展示内容
每本书以列表卡片展示：封面、书名、作者、价格、新旧程度、分类、校区、发布时间、想买人数、状态标签。

#### 筛选与排序
- 分类：全部、教材、小说、教辅、其他。
- 校区/学院/专业：三级级联筛选。
- 排序：最新发布、价格从低到高、价格从高到低、想买人数优先。
- 搜索：书名、作者、ISBN 模糊搜索。

#### 验收标准
- 默认只展示 `status=available` 的书籍。
- 分页每页 20 条，滚动加载或分页按钮均可。
- 搜索输入 300ms 防抖。
- 搜索关键词为空时恢复筛选列表。
- 响应式适配 360px 到桌面宽度。

### 5.2 书籍详情页
#### 展示内容
- 图片轮播：1-5 张。
- 书籍信息：书名、作者、ISBN、分类、价格、数量、新旧程度、描述。
- 位置信息：校区、学院、专业。
- 卖家信息：昵称、头像、发布时间、历史评价摘要。
- 交易状态：在售、已预定、已售罄、已下架。

#### 联系方式保护
- 详情初始接口不返回 `contactPlain`。
- 登录用户点击“我想买”并成功记录意向后，才能请求联系方式。
- 卖家本人可查看自己填写的联系方式。
- 接口日志不得记录完整联系方式。

### 5.3 发布新书
#### 录入模式
- ISBN 自动补全：输入 ISBN 或调用浏览器摄像头扫描条形码。
- 手动录入：当无 ISBN 或查询失败时使用。

#### 必填字段
- 书名
- 作者，未知可填“未知”
- 分类
- 价格
- 数量，默认 1
- 新旧程度
- 联系方式
- 至少 1 张图片

#### 图片规则
- 最多 5 张。
- 单张原图最大 5MB，前端压缩后建议小于 2MB。
- 支持 jpg、jpeg、png、webp。
- 服务端必须校验 MIME、扩展名、大小。
- 图片路径不得包含手机号、邮箱、用户 ID 等可识别信息。

#### 验收标准
- 前端校验失败时显示具体错误。
- 服务端二次校验，不信任前端传入字段。
- 发布成功后跳转“我的发布”或新书详情页。

### 5.4 表达想买
#### 规则
- 必须登录。
- 不能对自己发布的书表达想买。
- 同一用户对同一本书只能表达一次。
- 多个用户可表达想买，但是否锁定由库存和配置决定。
- 达到库存数量后可标记为 reserved，但最终 sold 仍由卖家确认。

#### 验收标准
- 成功后返回想买记录，并允许查看联系方式。
- 重复点击不会生成重复记录。
- 并发情况下不会超过库存上限，建议使用数据库事务或唯一约束。

### 5.5 交易留言板
#### 可见性
- 卖家可见。
- 已表达想买的买家可见。
- 其他用户不可见。

#### 规则
- 文本留言最多 500 字。
- 每本书最多 50 条留言，MVP 可按书维度共享留言板。
- 书籍 sold 或 removed 后关闭新留言。
- 留言列表按时间升序展示。

#### 验收标准
- 未授权用户请求留言接口返回 403。
- 留言成功后更新书籍 `lastMessageAt`。
- 我的发布列表能显示“有新留言”提示。

### 5.6 卖家确认售出
#### 规则
- 仅卖家可操作。
- 仅 available/reserved 状态可确认售出。
- 必须从已表达想买的用户中选择最终买家。
- 确认后写入 `buyerId`、`soldAt`，书籍状态变为 `sold`。

#### 验收标准
- 非卖家调用接口返回 403。
- 买家不在想买列表中时返回 400。
- 售出后从书城默认列表移除。
- 买卖双方获得评价入口。

### 5.7 双向评价
#### 规则
- 仅 sold 书籍的卖家和最终买家可评价。
- 每个交易方只能评价一次。
- 评分 1-5 星，评论最多 200 字。
- 标签最多 5 个。
- 前台展示匿名角色：买家/卖家，不展示账号 ID。

#### 验收标准
- 未成交用户不能评价。
- 重复评价被拒绝。
- 评价提交后详情页刷新评价列表。

### 5.8 举报
#### 举报对象
- 书籍
- 用户，v1 可只实现书籍举报，接口预留用户举报

#### 举报原因
虚假信息、诈骗、违禁品、骚扰、联系方式异常、其他。

#### 验收标准
- 登录用户可举报。
- 同一用户对同一目标只能举报一次。
- 举报状态默认 `pending`。
- 普通用户不能读取举报列表。

### 5.9 我的发布
#### 页面内容
- 用户资料卡片。
- 在售中、已售罄、已下架三个标签。
- 发布新书入口。
- 在售书籍可下架。
- 已售书籍展示成交时间和评价摘要。

#### 验收标准
- 只展示当前登录用户发布的书。
- 下架需要二次确认。
- 下架采用软删除，状态改为 `removed`，保留审计记录。

## 6. 非功能需求
### 6.1 性能
| 场景 | 指标 |
|------|------|
| 首屏可交互 | 移动端 4G 下 ≤ 2.5 秒 |
| 书城列表接口 P95 | ≤ 1.5 秒 |
| 详情接口 P95 | ≤ 1 秒 |
| 图片上传 | 单张压缩后 ≤ 2MB |
| API 分页 | 默认 20 条，最大 50 条 |

### 6.2 安全
- 密码、短信密钥、邮件密钥、对象存储密钥必须使用环境变量。
- 联系方式不得出现在列表接口、公开详情接口和前端初始 HTML 中。
- 使用 HTTPS。
- Cookie 设置 HttpOnly、Secure、SameSite=Lax 或 Strict。
- 对写接口做 CSRF 防护或使用 SameSite + CSRF token。
- 所有用户输入在展示时转义，避免 XSS。
- 上传文件不直接信任 Content-Type，服务端检查真实文件头。
- 关键操作写审计日志。

### 6.3 合规与隐私
- 提供用户协议和隐私政策页面。
- 明确说明平台不参与资金交易。
- 明确联系方式仅在用户表达想买后展示。
- 支持用户删除或注销账号的后续接口预留。

## 7. 优先级划分
| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 登录、会话、用户记录 | 无登录无法发布和交易 |
| P0 | 书城列表与详情 | 核心浏览链路 |
| P0 | 发布新书与图片上传 | 核心供给链路 |
| P0 | 想买与联系方式鉴权 | 交易撮合核心 |
| P0 | 卖家确认售出 | 防止恶意买家直接下架 |
| P0 | 我的发布和下架 | 卖家管理 |
| P1 | ISBN 自动补全 | 提升发布体验 |
| P1 | 留言板 | 降低联系摩擦 |
| P1 | 双向评价 | 建立信任 |
| P1 | 举报 | 风险兜底 |
| P2 | Web Push/邮件通知 | 提升触达 |
| P2 | 管理后台 | 运营治理 |
| P2 | 校园认证 | 提升可信度 |
