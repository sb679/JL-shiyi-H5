## 流程图说明
- `[页面]` 表示 H5 页面。
- `→` 表示用户操作。
- `→→` 表示系统自动执行。
- `✅` 表示成功终点。
- `❌` 表示失败路径。
- `⚠` 表示注意事项。

## 流程 1：首次访问与登录
```text
用户打开 H5 链接
  →→ App 请求 GET /auth/me
      ├── 已登录 → 获取用户信息 → 进入目标页面
      └── 未登录 → 以游客身份进入书城或详情页

用户点击发布/我想买/留言等登录操作
  → 跳转 [登录页]?redirect=当前页面
  → 输入手机号或邮箱
  → 勾选用户协议和隐私政策
  → 点击获取验证码
  →→ POST /auth/send-code
      ├── 成功 → 倒计时 60 秒
      └── 失败 ❌ → 显示限流或格式错误
  → 输入验证码
  → 点击登录
  →→ POST /auth/login
      ├── 成功 → 服务端设置会话 Cookie → 回到 redirect 页面
      └── 失败 ❌ → 提示验证码错误或过期
✅ 登录完成
```

注意：生产环境不得使用固定万能验证码；验证码和 token 不得输出到日志。

## 流程 2：浏览与搜索书籍
```text
[书城首页]
  →→ GET /configs/campus 加载校区配置
  →→ GET /books?page=1&pageSize=20&status=available
  ✅ 展示在售书籍列表

用户可执行：
  ├── 输入关键词 → 300ms 防抖 → GET /books?keyword=xxx
  ├── 点击分类 → 重置页码 → GET /books?category=textbook
  ├── 打开筛选 → 选择校区/学院/专业 → GET /books?campus=...
  ├── 切换排序 → GET /books?sort=price_asc
  ├── 下拉刷新或点击刷新 → 重新请求第一页
  ├── 滚动到底部 → 请求下一页
  └── 点击书籍卡片 → 跳转 [书籍详情] /books/:id
```

筛选级联：
```text
选择校区 → 清空学院和专业
选择学院 → 清空专业
选择专业 → 只改变专业筛选
点击重置 → 清空所有筛选条件
```

## 流程 3：发布新书（ISBN 自动补全）
```text
入口：[书城] 或 [我的发布] → 点击发布
  ├── 未登录 → 跳转登录 → 登录成功返回 [发布新书]
  └── 已登录 → 进入 [发布新书]

[发布新书]
  → 默认 ISBN 模式
  → 输入 ISBN 或点击扫码
      ├── 浏览器支持摄像头扫码 → 授权摄像头 → 扫描条形码
      ├── 浏览器不支持 → 显示手动输入提示
      └── 用户拒绝权限 → 手动输入
  → 点击查询
  →→ GET /isbn/:isbn
      ├── 成功 → 自动填充书名、作者、分类、封面参考
      └── 失败 ❌ → 提示手动录入

用户补充交易信息：
  → 价格、数量、新旧程度、联系方式
  → 校区/学院/专业
  → 描述
  → 上传 1-5 张图片
      → POST /uploads/presign
      → PUT 直传对象存储
      → 保存 objectKey
  → 点击发布
  → 前端校验
  →→ POST /books
      ├── 成功 → 跳转 /books/:id 或 /me/books
      └── 失败 ❌ → 展示具体字段错误
✅ 发布完成
```

## 流程 4：发布新书（手动录入）
```text
[发布新书]
  → 切换到手动录入
  → 填写书名、作者、ISBN 可选、分类
  → 填写价格、数量、新旧程度、联系方式、描述
  → 上传图片
  → 提交
  →→ POST /books
✅ 发布完成
```

手动模式仍必须满足：书名、作者、分类、价格、联系方式、至少一张图片。

## 流程 5：表达想买与查看联系方式
```text
[书籍详情]
  → 用户点击“我想买”
      ├── 未登录 → 跳转登录，登录后返回详情页
      └── 已登录 → 打开确认弹窗
  → 点击确认
  →→ POST /books/:id/express-interest
      ├── 成功 → 记录想买意向
      │        → 页面刷新权限 canViewContact=true
      │        → 自动请求或提示点击查看联系方式
      ├── 自己的书 ❌ → 提示不能购买自己发布的书
      ├── 重复想买 ❌ → 提示已表达过意向
      └── 库存已满 ❌ → 提示已被预定

用户点击“查看联系方式”
  →→ GET /books/:id/contact
      ├── 成功 → 显示联系方式 + 复制按钮
      └── 无权限 ❌ → 提示需先表达想买
✅ 买家获得联系方式，线下联系交易
```

隐私要求：联系方式不得通过书城列表或详情初始接口返回。

## 流程 6：交易留言
```text
前置条件：用户是卖家，或用户已表达想买

[书籍详情]
  →→ GET /books/:id/messages
      ├── 成功 → 展示留言列表
      └── 无权限 ❌ → 不展示留言板

发送留言：
  → 输入 1-500 字
  → 点击发送
  →→ POST /books/:id/messages
      ├── 成功 → 清空输入框 → 刷新留言列表
      ├── 书籍已售/下架 ❌ → 提示留言已关闭
      └── 超过 50 条 ❌ → 提示留言数量已达上限
```

售出后留言板关闭，但历史留言可供交易双方查看。

## 流程 7：卖家确认售出
```text
前置条件：当前用户是卖家，书籍状态为 available/reserved，且有想买用户

[书籍详情]
  → 卖家看到想买用户列表
  → 点击确认售出
  → 选择最终买家
  → 二次确认
  →→ POST /books/:id/confirm-sold { buyerId }
      ├── 成功 → 书籍状态变为 sold
      │        → 写入 buyerId 和 soldAt
      │        → 从书城默认列表移除
      │        → 买卖双方获得评价入口
      ├── 非卖家 ❌ → 403
      └── buyerId 不在想买列表 ❌ → 400
✅ 成交记录完成
```

## 流程 8：双向评价
```text
前置条件：书籍状态为 sold，当前用户是卖家或最终买家

[书籍详情]
  → 点击评价对方
  → 打开评价弹窗
  → 选择 1-5 星
  → 可选填写评论和标签
  → 点击提交
  →→ POST /books/:id/evaluations
      ├── 成功 → 关闭弹窗 → 刷新评价列表
      ├── 非交易方 ❌ → 403
      └── 重复评价 ❌ → 409
✅ 评价完成
```

展示策略：评价列表只展示角色“买家/卖家”、星级、标签、评论、时间，不展示用户 ID。

## 流程 9：我的发布与下架
```text
[我的发布]
  →→ GET /books?mine=true&status=available
  ✅ 展示在售书籍

用户切换标签：
  ├── 在售 → GET /books?mine=true&status=available
  ├── 已售 → GET /books?mine=true&status=sold
  └── 已下架 → GET /books?mine=true&status=removed

下架：
  → 点击某书“下架”
  → 二次确认
  →→ DELETE /books/:id
      ├── 成功 → 状态改为 removed → 刷新列表
      ├── 非卖家 ❌ → 403
      └── 已售书籍 ❌ → 不允许下架
✅ 下架完成
```

## 流程 10：修改个人资料
```text
[我的] → [个人资料]
  → 修改昵称
  → 上传头像
      → POST /uploads/presign
      → PUT 上传对象存储
  → 选择校区/学院/专业
  → 点击保存
  →→ PATCH /users/me
      ├── 成功 → 刷新当前用户信息
      └── 失败 ❌ → 显示字段错误
✅ 资料更新
```

## 流程 11：举报
```text
[书籍详情]
  → 点击页面底部“举报此书”
  → 选择举报原因
  → 可选填写补充说明
  → 提交
  →→ POST /reports
      ├── 成功 → 提示已收到举报
      ├── 未登录 ❌ → 跳转登录
      └── 重复举报 ❌ → 提示已举报过
✅ 举报记录进入 pending 状态
```

## 流程 12：分享
```text
[书籍详情]
  → 点击浏览器分享或复制链接
  → 系统分享 URL: https://example.com/books/:id
  → 对方打开链接
      ├── 公开信息可见
      └── 想买/联系方式/留言需登录
```

分享元信息：
- 标题：`【¥35】深入理解计算机系统 - JL拾遗`
- 描述：作者、校区、新旧程度。
- 图片：书籍封面。
- 不包含联系方式。

## 流程 13：异常与恢复
| 场景 | 用户提示 | 恢复动作 |
|------|----------|----------|
| 会话过期 | 登录已过期，请重新登录 | 跳转登录并保留 redirect |
| 网络断开 | 网络异常，请稍后重试 | 保留表单草稿 |
| 图片上传失败 | 第 N 张图片上传失败 | 支持重试或删除 |
| ISBN 查询失败 | 未查询到书籍信息，请手动录入 | 自动切换或提示手动模式 |
| 库存已满 | 这本书已被预定 | 禁用想买按钮 |
| 书籍已售 | 这本书已售出 | 展示售罄状态 |
