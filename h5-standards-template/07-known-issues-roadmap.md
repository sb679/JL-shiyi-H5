## 1. H5 版本关键风险
### R1. 联系方式泄露
| 项目 | 内容 |
|------|------|
| 严重程度 | 高 |
| 风险 | 联系方式属于敏感个人信息，若列表、详情、日志或前端缓存泄露，会造成骚扰和信任损失 |
| 缓解 | 联系方式加密存储；公开接口不返回；单独 `/contact` 鉴权；日志脱敏；不写入 localStorage |
| 验收 | 未表达想买的用户无法通过任何公开接口获得联系方式 |

### R2. H5 登录真实性弱于微信 openid
| 项目 | 内容 |
|------|------|
| 严重程度 | 中 |
| 风险 | 手机号/邮箱账号可能不是本校学生 |
| 缓解 | v1 允许开放注册；v1.1 增加校园邮箱认证或邀请码；管理员可封禁异常账号 |
| 验收 | 预留用户认证字段和状态，不阻塞 v1 MVP |

### R3. 验证码成本与滥用
| 项目 | 内容 |
|------|------|
| 严重程度 | 中 |
| 风险 | 短信/邮件验证码可能被刷导致费用或服务限制 |
| 缓解 | IP + identifier 限流；图形验证码可选；开发环境和生产环境验证码策略分离 |
| 验收 | 同一账号 60 秒内不能重复发送验证码 |

### R4. 图片上传成本和违规内容
| 项目 | 内容 |
|------|------|
| 严重程度 | 中 |
| 风险 | 大量图片造成存储和 CDN 费用，违规图片带来合规风险 |
| 缓解 | 前端压缩；服务端限制大小；对象存储生命周期；v2 接入内容审核 |
| 验收 | 单张图片超过限制时上传失败 |

### R5. 并发想买导致库存超卖
| 项目 | 内容 |
|------|------|
| 严重程度 | 中 |
| 风险 | 多个用户同时表达想买，可能超过库存上限 |
| 缓解 | 数据库事务 + 行级锁 + 唯一约束；不要只靠前端判断 |
| 验收 | 并发测试下 active interest 数量不超过 quantity |

### R6. H5 摄像头扫码兼容不稳定
| 项目 | 内容 |
|------|------|
| 严重程度 | 低 |
| 风险 | iOS/微信内置浏览器对 BarcodeDetector 和摄像头权限支持不一致 |
| 缓解 | 扫码只是增强功能，必须保留手动输入 ISBN |
| 验收 | 不支持扫码时用户仍能完成发布 |

### R7. SEO/分享图与 SPA 路由
| 项目 | 内容 |
|------|------|
| 严重程度 | 低 |
| 风险 | SPA 直接打开详情页 404，或分享时标题图片不准确 |
| 缓解 | Nginx fallback；必要时为详情页增加 SSR/预渲染元信息 |
| 验收 | 直接打开 `/books/:id` 正常展示 |

## 2. MVP 已知限制
| 编号 | 限制 | 影响 | 后续方案 |
|------|------|------|----------|
| L1 | 不做在线支付 | 用户需线下交易 | 保持信息撮合定位 |
| L2 | 不做强学生认证 | 可能有校外用户 | v1.1 校园邮箱/邀请码 |
| L3 | 留言不是完整 IM | 沟通能力有限 | v2 实时消息或站内通知 |
| L4 | 搜索使用数据库 | 数据量大后性能下降 | v2 接入 Meilisearch/Typesense |
| L5 | 举报无完整后台 | 运营处理低效 | v1.1 简易管理页 |
| L6 | 内容审核未自动化 | 违规内容需人工处理 | v2 接入图片/文本审核 |

## 6. 运营规则建议
### 6.1 禁止发布内容
- 盗版资料、违规出版物。
- 非书籍商品，除非后续版本扩展品类。
- 涉及诈骗、诱导转账、虚假联系方式的信息。
- 带有敏感个人信息的图片。

### 6.2 举报处理建议
| 举报数量 | 建议动作 |
|----------|----------|
| 1 次 | 进入 pending，管理员查看 |
| 2-3 次 | 标记高优先级，必要时临时隐藏 |
| 多次且理由一致 | 临时下架，通知卖家申诉 |

### 6.3 账号处罚建议
| 行为 | 处理 |
|------|------|
| 发布虚假信息 | 下架书籍，警告 |
| 多次虚假发布 | 限制发布 7 天 |
| 诈骗或严重骚扰 | 封禁账号，保留审计 |
