## 流程图说明

- `[页面]` 表示小程序页面
- `→` 表示用户操作触发

- `✅` 表示成功路径终点

## 流程 1：首次使用（登录）

```
用户打开小程序
  →→ app.js 初始化云开发 (wx.cloud.init)
  →→ 调用 getOpenid 云函数获取 openid
  →→ syncUserInfo(): 检查 users 集合
      ├── 新用户 →→ 创建记录 (nickname='拾遗用户', avatarUrl='')
      └── 老用户 →→ 更新 updateTime
  →→ globalData.isLoginReady = true
  ✅ 进入书城首页
```

**⚠ 当前问题**：
- `nickname` 恒为"拾遗用户"，`avatarUrl` 恒为空（微信已废弃 `wx.getUserInfo` 自动获取头像昵称能力）
- 用户无法修改头像和昵称（计划在后续版本中在"自己的书"页面添加修改入口）

## 流程 2：浏览与搜索书籍

```
[书城] 页面加载
  →→ loadCampusConfig() 获取校区/学院/专业配置
  →→ loadBookList(page=1, category='all', sortType='time')
  ✅ 展示在售书籍列表（每页20条）

用户操作选项：
  ├── 下拉刷新 → onRefresh() → 重置分页重新加载
  ├── 触底加载更多 → onReachBottom() → page++ 追加数据
  ├── 点击分类标签 → onCategoryChange() → 重置分页，按分类筛选
  ├── 切换排序 → onSortChange('time' ↔ 'price-asc')
  ├── 校区筛选 → onCampusFilterChange() → 级联清空学院/专业
  ├── 输入搜索 → onSearchInput() → 300ms防抖 → 关键词搜索
  └── 点击书籍卡片 → 跳转 [书籍详情]?id=xxx
```

**筛选器级联规则**：
```
校区选择 → 重置学院选择 → 重置专业选择
学院选择 → 重置专业选择（不重置校区）
专业选择 → 仅筛选专业
```

## 流程 3：发布新书（ISBN 录入）

```
入口：[书城] 或 [自己的书] → 点击发布按钮 → [发布新书]

步骤1：录入 ISBN
  ├── 手动输入 ISBN → 点击"查询书籍信息"
  ├── 点击扫码 → wx.scanCode(barCode) → 自动填入 ISBN
  └── 无 ISBN → 切换到"手动录入"模式（流程4）

步骤2：查询书籍信息
  →→ searchByISBN 云函数
      ├── 成功 → 自动填充：书名、作者、封面、分类
      │         └── 分类从 subjects 推断
      └── 失败 ❌ → 提示"查询失败，请手动录入"

步骤3：补充交易信息
  价格：[____] 元（必填）
  数量：[—] 1 [+]（默认1）
  新旧程度：[几乎全新 ▼]（必填）
  联系方式：[____________]（必填，微信/手机号）
  描述：[____________]（选填，最多200字）

步骤4：选校区（可选）
  校区 / 学院 / 专业：三级级联选择器
  [✓] 自动关联（开启后下次发布记住选择）

步骤5：上传图片
  点击"+" → wx.chooseMedia（最多5张）
  →→ 前端压缩（quality 80, maxLength 1920）
  →→ 上传到云存储 books/{timestamp}_{random}.jpg
  →→ 显示已上传图片（可点击X删除）
  校验：至少1张

步骤6：提交
  点击"立即发布" → 二次确认弹窗
  →→ validateForm() 前端校验
      ├── 失败 ❌ → 显示具体校验错误
      └── 通过 → publishBook 云函数
          ├── 云函数端二次校验
          ├── 写入 books 集合
          ├── 若开启自动关联 → syncUserCampus()
          └── ✅ 发布成功 → 自动跳转 [自己的书] 页面
```

**⚠ 注意事项**：
- Google Books API Key 硬编码在 searchByISBN 云函数中，有安全风险
- ISBN 查询超时 3 秒，失败后静默降级

## 流程 4：发布新书（手动录入）

```
入口：[发布新书] → 切换"手动录入"标签

步骤1：手动填写基本信息

  作者：[____________]（必填，1-50字）
  ISBN：[____________]（选填）
  分类：[ ▼]（默认"其他"）

步骤2-6：同流程3的步骤3-6
  补充交易信息 → 选校区 → 上传图片 → 提交
```

## 流程 5：表达购买意向（"我想买"）

```
前置条件：书籍状态为 available，未被锁定

[书籍详情] 页面
  ↓
用户点击"我想买"按钮
  ↓
二次确认弹窗："确定想买这本书吗？"
  ↓ 确认
expressInterest 云函数
  ├── 校验书籍存在 ✓
  ├── 校验状态为 available ✓
  ├── 校验未锁定 ✓
  ├── 校验不是自己的书 ✓
  ├── 校验未重复表达意向 ✓
  ├── 校验库存未耗尽 ✓
  ├── 将用户 openid 加入 interestedUsers
  ├── 若 interestedUsers.length >= quantity → isLocked = true
  └── ✅ 成功
      ├── 调用 getContact 云函数获取联系方式
      ├── 页面显示联系方式 + 复制按钮
      ├── 留言板出现（showMessageBoard = true）
      └── "我想买"按钮变为"查看联系方式"
```

**⚠ 注意**：
- v1 未实现微信订阅消息通知（需求中有规划）
- 同一本书的 interestedUsers 人数达到库存量时自动锁定
- 买家自己无法看到自己的书出现在"想买"列表中

## 流程 6：卖家确认售出

```
前置条件：当前用户是发布者，书籍状态为 available，有感兴趣用户

[书籍详情] 页面（仅卖家可见）
  ↓
点击"确认售出"按钮
  ↓
弹出 ActionSheet：从 interestedUsers 列表中选择最终买家
  ↓ 选择一个买家
二次确认弹窗："确定此书已售给 李四 吗？"
  ↓ 确认
confirmSold 云函数
  ├── 校验书籍存在 ✓
  ├── 校验调用者是发布者 ✓
  ├── 校验状态为 available ✓
  ├── 校验 buyerOpenid 在 interestedUsers 中 ✓
  ├── 更新 status = 'sold'
  ├── 记录 buyerOpenid, purchaseTime
  └── ✅ 成功
      ├── 书籍进入"已售罄"状态
      ├── 双方均可评价（canEvaluate = true）
      ├── 留言板关闭
      └── 底部操作栏显示"已售罄"
```

## 流程 7：双向评价

```
前置条件：书籍状态为 sold，当前用户是买家或卖家

[书籍详情] 页面
  ↓
点击"评价对方"按钮
  ↓
评价弹窗打开
├── 星级选择：[★][★][★][★][★]（1-5星）
├── 文字评价：[____________]（选填，最多100字）
├── 快捷标签：[爽快交易] [书况如实] [沟通顺畅]
├── [取消] → 关闭弹窗
└── [提交评价]
    ↓
submitEvaluation 云函数
  ├── 校验书籍 sold ✓
  ├── 校验角色（买家/卖家） ✓
  ├── 校验未重复评价 ✓
  ├── 写入 evaluations 集合
  └── ✅ 成功
      ├── 弹窗关闭
      ├── loadEvaluations() 刷新评价列表
      └── 评价以匿名形式展示（仅显示"买家"/"卖家"角色）

⚠ 当前问题：评价弹窗中的快捷标签仅展示，不可点击选择
```

## 流程 8：交易留言板

```
前置条件：用户是卖家，或已表达"想买"意向

[书籍详情] 页面 → 留言板区域自动显示
  ↓
loadMessages() → getMessages 云函数
  ├── 成功 → 显示留言列表（气泡样式）
  │          ├── 自己的消息：右对齐，蓝色气泡
  │          └── 对方的消息：左对齐，灰色气泡
  └── 空 → 显示"暂无留言，发送第一条消息吧~"

发送留言：
  → 输入文字 → 点击"发送"或回车
  → sendMessage 云函数
      ├── 校验书籍非 sold ✓
      ├── 校验调用者是交易方 ✓
      ├── 校验留言非空 ✓
      ├── 校验留言数 ≤ 50 ✓
      ├── 写入 messages 集合
      ├── 更新 books.lastMessageTime
      └── ✅ 成功 → 清空输入框 → 刷新留言列表

关闭条件：
  → 书籍售罄后留言板关闭，显示"交易已完成，留言板已关闭"

留言提示：
  → "自己的书"页面：lastMessageTime 非空时显示"· 有留言"提示
```

## 流程 9：下架书籍

```
[自己的书] → "在售中"标签
  ↓
点击某本书的"下架"按钮
  ↓
二次确认弹窗："确定下架这本书吗？"
  ↓ 确认
→→ 更新 books 集合 status = 'removed'
→→ 书籍从在售列表消失
✅ 下架成功
```

**注意**：已售罄的书籍不可下架。

## 流程 10：举报

```
[书籍详情] 页面
  ↓
滚动到页面最底部
  ↓
点击灰色极小字"举报此书"
  ↓
举报弹窗打开
├── 选择举报原因：[虚假信息] [诈骗] [违禁品] [骚扰] [其他]
│   └── 选中项高亮为主题色
├── 补充说明：[____________]（选填，最多200字）
├── [取消] → 关闭弹窗
└── [提交举报]
    ↓
submitReport 云函数
  ├── 校验未重复举报 ✓
  ├── 写入 reports 集合（status='pending'）
  └── ✅ 成功 → 提示"已收到举报"

⚠ v1 举报仅记录，系统不做任何自动处理
⚠ 目前仅支持举报书籍，不支持举报用户
```

## 流程 11：数据修复

```
[自己的书] → 空状态或无数据时
  ↓
看到提示："数据异常？点击修复旧数据"
  ↓
点击 → initDatabase 云函数
  ├── 创建/更新 campus 配置
  ├── 扫描并补全旧书籍缺失字段
  └── ✅ 显示修复结果（带 ✅/❌ 标记）
```

**适用场景**：
- 旧书籍记录缺少 status/isLocked/quantity 等字段
- 校区配置数据丢失

## 流程 12：分享

```
[书籍详情] 或 [书城] 页面
  ↓
点击右上角"···"
  ├── 发送给朋友 → onShareAppMessage()

  │   └── 分享路径：/pages/bookDetail/bookDetail?id=xxx
  └── 分享到朋友圈 → onShareTimeline()
      └── 分享标题：同 App Message
      └── 附带 query id=xxx
```
